## 1. Grid Initialization (The "Constructor")

In Next.js, you will likely use a `useState` hook to hold your grid. Following the task requirements, your initialization logic must:

* 
**Dimensions:** Create a 2D array (matrix) that is **two rows and two columns larger** than the requested size (e.g., for a 10x10 game, create a 12x12 matrix).


* 
**The Border:** Set all "outer" cells (index 0 and index $N+1$) to **0**.


* 
**Randomization:** Fill the "inner" cells randomly with **0** (empty) or **1** (cell).



## 2. The Evolution Logic (`KovetkzoAllapot`)

This is the "brain" of the simulator. You must determine the state of the next generation without modifying the current one mid-calculation.

### Neighbor Counting

For every **inner cell**, you must check its 8 neighbors (including diagonals).

* A neighbor is any cell with a value of **1**.


* **Crucial Rule:** You must use a **buffer (copy)** of the matrix to calculate the new state. The original `Matrix` should only be updated once all new values are determined.



### Survival & Birth Rules

Apply these three rules to each inner cell:

1. 
**Survival:** A cell stays alive if it has **2 or 3** neighbors.


2. 
**Death:** A cell dies if it has **fewer than 2** (loneliness) or **more than 3** (overpopulation) neighbors.


3. 
**Birth:** An empty cell becomes alive if it has **exactly 3** neighbors.

## 3. The Simulation Loop (`Run`)

In a console app, you would use a loop with a sleep timer. In Next.js, you should use `useEffect` combined with `setInterval`.

* 
**Timing:** Set the interval to **500 milliseconds**.


* 
**Trigger:** The loop should run repeatedly until a "stop" condition is met (the task mentions "until a keypress," which in a browser could be a button toggle or a keyboard event listener).
